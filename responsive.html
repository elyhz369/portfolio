REDESIGN: Toronto Cupcake


Redesigning the homepage for a bakery store.


Have you ever visited a website and find it really hard to navigate? A good website should be user-centric with the following properties: First, new users should easily understand what to do (learnability). In addition, it should be intuitive to recall how to use and what to do (memorability). Futhermore, typical tasks should be done quickly (efficiency). Keeping these qualities in mind, this project will focus on visually redesigning a website and making it more user friendly.



PART I - Identifying Problems


This is a preview of the original website. Toronto Cupcake is a bakery store providing cupcake delivery services.

The site I picked is Toronto Cupcake. I chose this site because I used it to order cupcakes for my friend, and the website design was notoriously known in my friend group. The problems are as follows:


  1. LOW USABILITY
    • Navigation bar located at the bottom with a very small font size. It’s hard for users to even notice it.
    • Inconsistent hierarchy. Delivery time, cupcake ingredients, and branding slogan should not be placed in the same box.

  2. LOW LEARNABILITY
    • Bad image selection - a bunch of boxes in the page may confuse the users that this is a package design site. A cupcake store should put an image of delicious and well-decorated cakes in their main page to appeal to the customers.
    • Too much information in the folded menu. The main purpose of the website for users is ‘see the menu and place order.’ ‘Events’, ‘Resources’ and ‘Giving back’ should be placed on social media accounts or under the same page.

  3. LOW MEMORABILITY
    • Hard to locate the ‘order’ button. As said above, users are here to make orders. The order button must be on a different hierarchy and easily noticed, otherwise it's hard to remember and users have to spend a lot of time finding it each time.

On the good side, the website has good accessibility. There are no errors detected by WebAIM WAVE.

Notes for Coding
  • Create a div for the background image and its overlaying content.
  • Set left & right margin to 5%.
  • Make sure background-attachment is fixed.
  • Create a grid to position items (such as logo and title) on top of the background image.
  • Alternatively, use 'position: absolute' and position the items.
  • Use flex-wrap for contents below.

    Notes for Coding
  • When device width is less than 1100px, use media query and change the navbar to a hamburger menu
  • Everything else in the div containing background image should be inherited if grid was used.
  • Restyle the 'best seller' div matching the design in prototype.
  • Within @media(max-width: 1100px), show the restyled best seller div and hide the best seller div for desktop.
  • Resize text if there're overlaps.

    Notes for Coding
  • When device width is less than 700px, resize the div containing background image, also reset the grid.
  • Similarly, when device width is less than 400px, resize the div and reset the grid.
  • The grid should have 5 rows and 3 columns.
  • Put logo, menu, and cart in each column at first row. The title 'TORONTO CUPCAKE' should be in the 2nd column and take rows 2-4.
  • Restyle best seller, show the restyled div, and hide the best seller div for tablet.
